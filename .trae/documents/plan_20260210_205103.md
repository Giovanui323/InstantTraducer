I will optimize the application startup sequence in `electron/main.js` to prevent the Deep Health Check and other maintenance tasks from delaying the UI initialization.

The current implementation awaits `performDeepHealthCheck()` (and other tasks) *before* calling `createWindow()`. This delays the window creation by several seconds (7s as reported), which the user perceives as slow group loading.

My plan is to:

1. **Refactor** **`electron/main.js`**:

   * Move `createWindow()` to be called immediately after essential setup (settings & health monitoring), but *before* the heavy maintenance tasks.

   * Group the maintenance tasks (`recoverTemporaryFiles`, `performDeepHealthCheck`, `autoCleanupTrash`, `performConsolidation`) into a new async function `performStartupMaintenance`.

   * Call `performStartupMaintenance()` without awaiting it, allowing it to run in the background while the UI loads.

   * Add logging to track the start and end of this background maintenance.

This change will allow the application window to appear immediately. The `load-groups` IPC request (triggered by the UI) will be processed in parallel with the background health checks, significantly improving the perceived startup time.
