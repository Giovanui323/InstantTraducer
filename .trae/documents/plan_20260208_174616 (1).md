I understand the core issue: the log files are not automatically recreated ("reforming") after deletion because the application still holds an open connection to the deleted files.

I will fix this by forcing the application to close these connections during cleanup. This ensures that the next time the application tries to log something, it will detect that no connection exists and will **automatically create a new log file**.

1.  **Modify `electron/logHandlers.js`**:
    *   Update `executeLogCleanup` to **always** call `closeAllStreams()`.
    *   **Effect**: This releases the file handles. The next log write will trigger `ensureStream`, which will check the file system, see the file is missing, and **create it again**. This guarantees the files will "reform".

2.  **Modify `src/components/LogViewer.tsx`**:
    *   Update the "Pulisci Tutto" button to call `cleanupOldLogs(-1)` (delete all).
    *   Update the tooltip to "Elimina tutti i file di log".

This directly addresses the problem of files not reappearing after deletion.
