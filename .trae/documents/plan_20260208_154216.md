I have identified the cause of the "Attempted to register a second handler" error.

**The Problem:**
The application calls `setupPackageHandlers` (and others) inside `createWindow()`. While the code attempts to prevent re-registration using a `handlersRegistered` flag, this flag seems to be getting out of sync with the actual state of `ipcMain` handlers, likely due to how the main process or modules are being managed during window re-creation (e.g. after a window close and re-open on macOS). This causes the application to try to register `ipcMain` handlers that already exist, leading to a crash.

**The Solution:**
We need to explicitly remove any existing handlers for the specific channels before registering them. This ensures that even if the `handlersRegistered` flag is false, we won't crash by trying to double-register.

**Plan:**
1.  **Modify `electron/packageLogic.js`**:
    -   In `setupPackageHandlers`, add `ipcMain.removeHandler(...)` for `export-project-package` and `import-project-package` before registering them.
2.  **Modify `electron/pdfHandlers.js`**:
    -   In `setupPdfHandlers`, add `ipcMain.removeHandler(...)` for `export-translations-pdf` before registering it.
3.  **Modify `electron/projectHandlers.js`**:
    -   In `setupProjectHandlers`, add `ipcMain.removeHandler(...)` for all project-related IPC channels (e.g., `open-file-dialog`, `save-translation`, etc.) and `ipcMain.removeAllListeners` for `save-translation-requested` before registering them.

This approach makes the handler registration idempotent and robust against state inconsistencies.
