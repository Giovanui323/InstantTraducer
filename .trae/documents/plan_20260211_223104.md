I will fix the race condition causing the "Automatic Rename Error" and subsequent translation pause.

The issue occurs because the application attempts to rename the project file before it has been fully written to the disk. The current `blockSave` mechanism inadvertently cancels any pending save operations for the file, leading to a "File not found" error during the rename process.

To fix this, I will ensure that any pending saves are flushed (forced to complete) before blocking saves and attempting the rename.

### Plan:

1.  **Modify `src/App.tsx`**:
    *   In the `continueUploadWithLanguage` function (Automatic Rename logic):
        *   Insert `await library.flushSaves(fileId);` immediately before `library.blockSave(fileId, 8000);`.
    *   In the `RenameModal` `onRename` handler (Manual Rename logic):
        *   Insert `await library.flushSaves(oldId);` immediately before `library.blockSave(oldId, 8000);`.

This change ensures the file exists on disk before the rename operation begins, preventing the error and allowing the translation to start automatically as expected.
