I will optimize the initialization logic in `src/hooks/useAppLibrary.ts` to prevent the double loading of the project library at startup.

**Problem Analysis:**
The user logs show `refreshLibrary` is called twice within 48ms on startup. This is caused by the `useEffect` in `useAppLibrary` firing twice, likely due to React StrictMode (development behavior) or a rapid component remount. This causes unnecessary disk I/O and potential UI flickering.

**Proposed Changes:**

1. **Modify** **`src/hooks/useAppLibrary.ts`**:

   * Introduce a `useRef` named `hasInitialized` to track if the initial load has already been triggered for the component instance.

   * Update the `useEffect` responsible for the initial load (lines 136-139) to check `!hasInitialized.current` before calling `refreshLibrary()` and `loadGroups()`.

   * Set `hasInitialized.current = true` immediately after the check.

**Verification:**

* This change ensures that `refreshLibrary` and `loadGroups` are called exactly once per component mount lifecycle, effectively ignoring the second invocation that occurs in React Strict Mode or rapid remount scenarios.

* Manual calls to `refreshLibrary` (e.g., via the "Refresh" button) will still work as they bypass this `useEffect`.

**Code Change:**

```typescript
  const hasInitialized = useRef(false);

  useEffect(() => {
    if (!hasInitialized.current) {
      hasInitialized.current = true;
      void refreshLibrary();
      void loadGroups();
    }
  }, [refreshLibrary, loadGroups]);
```
