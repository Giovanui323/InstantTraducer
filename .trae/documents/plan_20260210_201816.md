Ecco il piano per risolvere il bug critico del "Blacklisting Permanente" e la perdita di dati durante la rinomina fallita.

### Analisi del Problema
Il bug è causato da una mancata gestione degli errori durante il processo di rinomina automatica:
1.  Il sistema chiama `blockSave(oldId)` per impedire salvataggi durante la rinomina.
2.  Tenta la rinomina tramite `window.electronAPI.renameTranslation`.
3.  Se la rinomina fallisce (es. errore `ENOENT` perché il file non viene trovato), il codice passa al blocco `catch` o `else`.
4.  **Il problema critico:** In questi casi di fallimento, **non viene mai chiamato `unblockSave(oldId)`**.
5.  Di conseguenza, l'ID rimane nella "blacklist", impedendo qualsiasi salvataggio futuro dei progressi (traduzioni pag 2, 3, 4, 5) per quella sessione.

### Piano di Intervento

Modificherò il file `src/hooks/useProjectManagement.ts` per garantire che il blocco dei salvataggi venga rimosso in *qualsiasi* scenario (successo o fallimento).

#### 1. Modifica `scanAndRenameOldFiles` e `scanAndRenameAllFiles`
Aggiornerò entrambe le funzioni per includere una gestione robusta dello sblocco (`unblockSave`) nei rami di errore.

**Logica Corrente (Semplificata):**
```typescript
blockSave(fileId);
try {
  const res = await rename(...);
  if (res.success) {
    // ...
    unblockSave(newId); // Sblocca SOLO il nuovo ID se tutto va bene
  }
} catch (e) {
  // NESSUNO SBLOCCO QUI -> ID bloccato per sempre
}
```

**Logica Corretta:**
```typescript
blockSave(fileId);
try {
  const res = await rename(...);
  if (res?.success) {
    // Successo: sblocca il nuovo ID
    unblockSave(res.fileId || newId);
    // ... logica aggiornamento stato ...
  } else {
    // Fallimento (API restituisce false): ripristina il vecchio ID
    unblockSave(fileId); 
    skipped++;
  }
} catch (e) {
  // Errore (es. ENOENT): ripristina il vecchio ID
  unblockSave(fileId);
  log.error(...);
  skipped++;
}
```

### Benefici
- **Prevenzione Data Loss:** Se la rinomina fallisce, l'utente potrà continuare a salvare i progressi sul file originale.
- **Resilienza:** Il sistema gestirà errori imprevisti (come file mancanti o permessi negati) senza bloccare l'intera applicazione.
