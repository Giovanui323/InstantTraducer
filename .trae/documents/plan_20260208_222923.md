### 1. Eliminate "Single Slot Policy" (Enable Parallel Translation)
**File:** `src/services/translation/TranslationManager.ts`
- **Action:** Modify `notifyActiveTranslationStarted`.
- **Change:** Remove the logic that pauses/stops the background job. This will allow the Background Job (old project) to continue running even while you work on the Active Project (new project).

### 2. Fix State Wipe & Data Loss (TranslationsCount: 0)
**File:** `src/App.tsx` (inside `handleOpenProject`)
- **Root Cause:** When switching projects, the app moves the old project to background and *immediately* resets the memory (`resetProjectState`) before ensuring the latest progress is saved to disk. The Background Runner then reads "empty" or partial data from disk, overwriting your progress.
- **Action:** 
    1.  Call `await library.flushSaves()` (or equivalent) **before** moving the project to background and resetting state.
    2.  Ensure `translationManager.requestBackground(...)` receives the fully up-to-date `translationMap` and `pdfDoc`.
- **Outcome:** The Background Runner will start with the correct, saved state.

### 3. Fix IPC Error (Image Save Failure)
**File:** `src/services/translation/TranslationJobRunner.ts`
- **Root Cause:** The runner tries to save a page image (`save-project-image`) via IPC, but sometimes the image data is missing (undefined/null), causing the "Nessun dato immagine fornito" error.
- **Action:** Add a validation check in `saveSourceForPage` (or before calling it). If `dataUrl` is missing, skip the save operation and log a warning instead of crashing the IPC channel.
- **Outcome:** Prevents the "Background stopped" error caused by image save failures.

### 4. Fix Infinite Loading / Race Condition
**File:** `src/App.tsx`
- **Action:** Strengthen the `openingProjectRef` guard in `handleOpenProject`.
- **Change:** Ensure that if a project load is already in progress for a specific ID, subsequent requests are strictly ignored until the first one completes or fails.
