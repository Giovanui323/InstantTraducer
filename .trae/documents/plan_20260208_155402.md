I will implement a fix for the race condition where the translation doesn't start automatically because the settings (and thus the API configuration status) haven't finished loading when the project is opened.

1.  **Modify `src/hooks/useAiSettings.ts`**:
    *   Add a `settingsLoaded` state variable, initialized to `false`.
    *   Set `settingsLoaded` to `true` after the settings loading attempt (whether successful or failed) in the `useEffect`.
    *   Return `settingsLoaded` from the hook.

2.  **Modify `src/App.tsx`**:
    *   Destructure `settingsLoaded` from the `useAiSettings` hook.
    *   Add a `useEffect` that listens for changes in `settingsLoaded` and `isApiConfigured`.
    *   Inside this effect, if `settingsLoaded` is true, `isApiConfigured` is true, the app is currently paused, and a project is open (not in manual/consultation mode), automatically set `isPaused` to `false` to start the translation. This ensures that if the settings load "late" (after the project is opened), the translation will still kick off automatically.

This approach resolves the user's issue where they had to manually click "Start" because the app initially thought the API wasn't configured due to the settings still loading.
