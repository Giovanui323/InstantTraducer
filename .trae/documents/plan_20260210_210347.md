I will implement fixes to resolve the "Blocked Queue", "Dependency Stall", and "Shutdown Flush" issues.

### 1. Fix Queue Dependency Blocking (The "Zombie Page" Issue)
**Problem:** Page 3 stays blocked because it thinks Page 2 is still in the queue (`queued.has(prevPage)` is true), even though Page 2 has finished (`Status: OK`) and is no longer in the actual processing queue (`translationQueueRef`). This synchronization drift between the `queued` Set and `queue` Array causes a deadlock.

**Solution:**
- **Modify `checkDependency` in `useAppTranslation.ts`**:
  - When checking if a dependency is queued (`queued.has(prevPage)`), add a secondary verification: check if `prevPage` is *actually* present in the `queue` array.
  - If the page is in the `queued` Set but NOT in the `queue` Array, treat it as a "Zombie" (already processed) and allow the current page to proceed.
  - This ensures the `queue` Array (the source of truth) overrides the `queued` Set optimization.

### 2. Improve Queue Processing & Diagnostics
**Problem:** The system logs "All pages blocked" but doesn't self-correct or provide details on *why* (e.g., "Page 3 blocked by Page 2").

**Solution:**
- **Modify `useTranslationQueue.ts`**:
  - Enhance the "All pages blocked" log to show exactly which page is blocking the head of the queue.
  - Add a **Sanity Check** when the queue is stuck: If `queuedPagesRef` contains pages that are missing from `translationQueueRef` and not active/in-flight, automatically remove them from the Set to self-heal the queue.

### 3. Fix Shutdown Flush & Handshake
**Problem:** "Flush required two attempts". The `stop()` method might be hanging or failing on the first attempt if a save is in progress or if `flushPendingSave` encounters an error, causing the shutdown handshake to time out or require a retry.

**Solution:**
- **Modify `TranslationJobRunner.ts`**:
  - Make `stop()` **idempotent**: If called multiple times, ensure it handles the state gracefully without re-triggering logic.
  - Wrap `flushPendingSave()` with a timeout race in `stop()` to prevent it from hanging the application close indefinitely if the disk write stalls.
  - Ensure `pendingSaveData` is cleared *before* the async save operation to prevent double-saving or race conditions during rapid shutdowns.
