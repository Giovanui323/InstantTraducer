I will implement a persistent visual indicator (a red dot) in the top-left corner of the application to signal severe issues.

### Implementation Plan

1.  **Modify `src/App.tsx`**:
    *   **Add State**: Introduce a new state variable `internalSevereError` to track critical errors reported by the translation engine.
    *   **Update Handler**: Modify the existing `onSevereError` callback to set this new state to `true` when a severe error occurs (e.g., translation failure after max retries).
    *   **Logic**: Determine the visibility of the indicator based on two conditions:
        *   `internalSevereError` is true.
        *   `corruptedPages` set is not empty (indicating rendering failures).
    *   **UI Component**: Insert a `div` element representing the red dot:
        *   **Position**: Absolute positioning at `top-2 left-2`.
        *   **Style**: Red background (`bg-red-600`), rounded (`rounded-full`), pulsing animation (`animate-pulse`), and a glow effect (`shadow`).
        *   **Z-Index**: High z-index (`z-[9999]`) to ensure it stays on top of the Header and other elements.
        *   **Tooltip**: Add a `title` attribute to explain the indicator on hover.

### Expected Behavior
*   When a severe error occurs (e.g., translation stopped due to repeated failures) or a page is corrupted, a pulsing red dot will appear in the top-left corner.
*   The dot will remain visible as long as the error state persists or corrupted pages exist.
