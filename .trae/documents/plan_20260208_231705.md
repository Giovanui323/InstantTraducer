# Critical Bug Fixes Plan

I have identified the root causes for the reported issues and prepared a fix plan.

## 1. Fix "Invalid Page Request" (Rendering Error)

**Problem:** The application attempts to render pages that do not exist (e.g., page 11 of a 10-page PDF) or accesses the PDF document after it has been destroyed.
**Solution:**

* **File:** `src/hooks/usePdfDocument.ts` and `src/services/translation/TranslationJobRunner.ts`

* **Action:**

  * Add bounds checking: Ensure `pageNumber` is always between `1` and `pdfDoc.numPages` before calling `getPage`.

  * Add null checks: Verify `pdfDoc` exists before attempting any operation, to prevent crashes when switching projects rapidly.

## 2. Fix "ID Mismatch" (Data Corruption Risk)

**Problem:** The system sometimes uses the "Active Project ID" to save data for a background project, causing the "ID Mismatch" error and potential data overwrites.
**Solution:**

* **File:** `src/hooks/useAppLibrary.ts`

* **Action:**

  * Modify `updateLibrary` to strictly prioritize the `fileId` contained in the data payload over the currently active session ID.

  * Add a safety check to prevent saving if `fileId` cannot be reliably determined from the payload itself.

## 3. Fix Null Pointer Exceptions (`sendWithPromise`)

**Problem:** Crashes occur when the PDF worker is terminated (e.g., during "Retry" or project switch) but a render task is still pending.
**Solution:**

* **File:** `src/hooks/usePdfDocument.ts`

* **Action:** Wrap `getPage` and render calls in try-catch blocks that specifically ignore "Worker Destroyed" errors, and ensure cleanup functions cancel pending tasks immediately.

  <br />

## 5. Fix File Inconsistency (Delete Loop)

**Problem:** After deleting a project, the app tries to reload it before the UI updates, causing "File not found" errors.
**Solution:**

* **File:** `src/hooks/useAppLibrary.ts`

* **Action:** Update `deleteProject` to **immediately remove** the project from the local `recentBooks` state *before* triggering the library refresh. This breaks the loop of trying to access a deleted file.

***

I will implement these changes sequentially, starting with the critical rendering and ID safety fixes.
