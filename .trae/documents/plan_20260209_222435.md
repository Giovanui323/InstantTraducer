I will optimize the translation pipeline to move the CPU-intensive post-processing (text cleaning and formatting) *after* the Quality Check. This ensures that resources are not wasted on text that is about to be discarded and re-translated.

### Technical Plan

1.  **Refactor Text Cleaning (`src/services/textClean.ts`)**
    *   Split the current `cleanTranslationText` function into two parts:
        *   `stripPreamble`: A lightweight function to remove meta-text (e.g., "Ecco la traduzione...") which is necessary before Quality Check.
        *   `cleanTranslationText`: The full post-processing function (includes `stripPreamble` and the heavier regexes for formatting/OCR fixes).

2.  **Update Gemini Service (`src/services/geminiService.ts`)**
    *   Modify `translateWithGemini` to accept a new optional parameter `skipPostProcessing`.
    *   If `skipPostProcessing` is `true`, perform only the lightweight `stripPreamble`.
    *   If `false` (default), perform the full `cleanTranslationText` (preserving existing behavior for other callers).

3.  **Update AI Service (`src/services/aiService.ts`)**
    *   Propagate the `skipPostProcessing` parameter through the `translatePage` wrapper function.

4.  **Optimize Translation Executor (`src/services/translation/TranslationExecutor.ts`)**
    *   Update `executePageTranslation` to:
        1.  Call `translatePage` with `skipPostProcessing: true`.
        2.  Update the UI (`setTranslationMap`) with the *draft* text immediately (so the user sees progress).
        3.  Run the **Quality Check** (`verifyAndMaybeFixTranslation`) on the draft text.
        4.  **Conditional Post-processing & Saving**:
            *   **If Quality Check Passes** (or is skipped/minor issues): Run the full `cleanTranslationText`, update the UI with the final polished text, and *then* save to disk (`updateLibrary`).
            *   **If Quality Check Fails (Severe)**: Skip the expensive post-processing and skip saving to disk (since a retry will likely follow), saving both CPU and I/O.

### Benefits
*   **CPU Savings**: Eliminates regex/formatting overhead on rejected translations.
*   **I/O Savings**: Avoids writing "bad" translations to disk that are immediately overwritten by retries.
*   **User Experience**: The UI still updates immediately with the draft translation, then refines it after verification.
