**Implementation Details:**
In `src/hooks/useAppTranslation.ts`, I will update the `MAX_CONCURRENT_TRANSLATIONS` calculation passed to `useTranslationQueue`:

```typescript
// Current
MAX_CONCURRENT_TRANSLATIONS: Math.max(1, Math.min(10, Number(aiSettings.translationConcurrency) || 2)),

// Proposed
MAX_CONCURRENT_TRANSLATIONS: (aiSettings.sequentialContext ?? true)
  ? 1 
  : Math.max(1, Math.min(10, Number(aiSettings.translationConcurrency) || 2)),
```

This change ensures that if `sequentialContext` is true (or undefined, defaulting to true), the concurrency is strictly 1, preventing the scheduler from attempting to start parallel tasks that would immediately block due to dependencies. If `sequentialContext` is false, it respects the user's concurrency setting (clamped between 1 and 10).
