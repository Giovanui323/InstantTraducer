I will implement a strict "Default OFF" policy for verbose logs to ensure a cleaner console output.

### Objective
Ensure `verboseLogs` defaults to **FALSE** and only activates when explicitly enabled by the user. Eliminate the discrepancy where logs appear despite the setting being disabled.

### Implementation Steps

#### 1. Enforce "Off by Default" in `src/services/logger.ts`
- **Current Logic**: `!== '0'` (Evaluates `null` as `true` -> **ON** by default).
- **New Logic**: `=== 'true'` (Evaluates `null` as `false` -> **OFF** by default).
- This ensures that if the setting hasn't been set, the logs are silent.

#### 2. Connect Settings to Logger in `src/App.tsx`
- **Sync Mechanism**: Update the existing `useEffect` to write the `verboseLogs` setting to `localStorage`.
  - If `aiSettings.verboseLogs` is `false`, it will write `'false'` to storage.
  - This guarantees the global logger stays in sync with the UI switch.
- **Remove Stale State**: Delete the isolated `verboseLogs` variable that reads from storage only once at startup.
- **Direct Access**: Update `appendPageConsole` to read `aiSettings.verboseLogs` directly, so it reacts immediately to changes.

### Result
- **Default State**: Application starts with clean logs (verbose mode OFF).
- **Control**: Toggling the switch in Settings gives immediate, full control over both UI and background logs.
