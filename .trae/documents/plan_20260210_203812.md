I will enhance the startup diagnostics to correctly detect and report system inconsistencies, specifically addressing the "False Positives" and "Orphaned Assets" issues.

### 1. Enhanced Inconsistency Tracker (`electron/state.js`)
I will add new metrics to the `inconsistencyTracker` to track:
- `crashDetected`: To indicate if the previous session terminated abnormally.
- `orphanedAssets`: To track asset folders that have no corresponding project file.
- `missingAssets`: To track project files that are missing their asset folder.

### 2. New Diagnostic Logic with Deep Logging (`electron/fileUtils.js`)
I will implement a new function `performDeepHealthCheck` that goes beyond simple `.tmp` file recovery and includes detailed logging as requested:
- **Crash Detection**: Implement a "dirty flag" mechanism. Create a `.running` file on startup and remove it on clean shutdown. If it exists at startup, flag a `crashDetected` and log a warning: `Unclean shutdown detected`.
- **Orphaned Asset Detection**:
    - Scan the `translations` directory to build a list of valid Project IDs.
    - Scan the `assets` directory to list all asset folders.
    - Identify asset folders that do not match any valid Project ID.
    - **Deep Logging**: For every orphaned folder found, log a specific warning: `Found orphaned asset folder: {path}`.
    - Update the `orphanedAssets` counter.
- **Missing Asset Detection**:
    - For each valid Project ID, verify that the corresponding `assets` folder exists.
    - **Deep Logging**: If missing, log a specific warning: `Missing asset folder for project: {projectId}`.
    - Update the `missingAssets` counter.

### 3. Startup Integration (`electron/main.js`)
I will update the `app.on('ready')` sequence to:
1.  Check for the "dirty flag" (Crash Detection) immediately.
2.  Run `performDeepHealthCheck` alongside `recoverTemporaryFiles`.
3.  Include the new metrics in the final "Library Health Report" log.
4.  Manage the lifecycle of the `.running` file (create on start, delete on quit).

This approach turns the Health Report into a true diagnostic tool that reflects the actual state of the disk and the reliability of the previous session, fulfilling the user's request to eliminate false positives and providing the deep logging requested.
