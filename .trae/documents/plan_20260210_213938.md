### **Planned Changes**

1.  **Prevent Queue Saturation (Deduplication Strategy)**:
    *   Modify `updateLibrary` to use an **"Upsert"** logic instead of blindly appending to the queue.
    *   If a save request for a `fileId` already exists in the queue, it will be **replaced** (updated) with the new data and timestamp, rather than creating a duplicate entry.
    *   This ensures the queue size roughly matches the number of *active files* rather than the number of *edit events*, drastically reducing memory usage.

2.  **Garbage Collection for Stale Requests**:
    *   Define a `MAX_REQUEST_AGE` (e.g., 5 minutes) in `processSaveQueue`.
    *   Automatically discard any pending requests that are older than this limit to prevent accumulation of "zombie" tasks.

3.  **Early Filtering of Blocked Files**:
    *   Although `updateLibrary` checks for blocked files, I will ensure `processSaveQueue` performs a **pre-check** before even attempting to group or process requests.
    *   Verify `cancelPendingSaves` is robust ensuring immediate removal of blocked files from the pending queue.
