I have analyzed the reported issues and identified the root causes and necessary fixes.

### 1. Concurrency Discrepancy (Active: 1/1)
- **Cause**: The `sequentialContext` setting (enabled by default) enforces strict sequential processing in `useAppTranslation.ts`. The `checkDependency` function blocks subsequent pages even if `translationConcurrency` is set to 2.
- **Resolution**: This is expected behavior for Narrative Continuity. I will clarify this in the documentation or code comments. No code change is needed unless you want to override this safety mechanism.

### 2. Context Synchronization (Page 3 ignores Page 2 retry)
- **Cause**: A race condition in `useAppQuality.ts`. When a retry succeeds, the `verificationMap` state is updated, but the `verificationMapRef` (used by the Translation Queue) is only updated in a `useEffect` hook. The queue proceeds to Page 3 *before* this effect runs, causing Page 3 to see the old "SEVERE" status of Page 2.
- **Fix**: Manually update `verificationMapRef.current` immediately inside `processVerificationInternal` to ensure the next queued item sees the latest state.

### 3. Circuit Breaker / Queue Reset
- **Cause**: The log "Queue and Circuit Breaker reset" comes specifically from `resetQueue`, which is **only** called by `resetProjectState` in `App.tsx`. This function also clears `pdfDoc` (destroying the PDF in memory).
- **Insight**: This is likely triggered by an accidental "Project Open/Upload" event or a "Close Session" action, not by the Circuit Breaker itself. The Circuit Breaker's internal reset logs "Circuit breaker reset for page X".
- **Action**: I will add more descriptive logging to `resetProjectState` to identify *what* triggered the reset (e.g., "User Action", "File Load", etc.).

### 4. Quality Check Severity
- **Cause**: The prompt in `verifierPrompts.ts` does not explicitly categorize "untranslated text" as a `severe` error, leading the AI to sometimes classify it as `minor`.
- **Fix**: I will update `verifierPrompts.ts` to explicitly state that "Text left in original language" must be classified as `severity: "severe"`.
