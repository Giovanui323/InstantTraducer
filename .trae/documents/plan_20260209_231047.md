I have identified the cause of the "Punitive Image Compression" issue.

### The Problem
When the system processes a page for the first time, it renders a high-quality image in memory for the AI. However, **simultaneously**, it aggressively compresses and downscales this image (to 1600px max edge and 74% quality) before saving it to the disk cache.

When a "Retry" occurs (either automatically or manually), the system prioritizes loading the image from the disk cache to save time. Consequently, the Retry attempt loads this **degraded, blurry version** (approx. 269KB) instead of re-rendering the high-quality original (approx. 480KB). This causes the AI to fail or hallucinate because it's reading a compressed artifact.

### The Solution
We need to upgrade the cache settings to ensure the saved image maintains high fidelity for future retries.

**Plan:**
1.  **Update `src/constants.ts`**:
    *   Increase `PAGE_CACHE_JPEG_QUALITY` from **0.74** (74%) to **0.88** (88%).
    *   Increase `PAGE_CACHE_MAX_EDGE` from **1600** to **2500** pixels (better support for A4 at 300DPI).
    *   *Note: This ensures that the cached image on disk is nearly identical to the high-quality in-memory render.*

2.  **Verify `src/services/translation/TranslationExecutor.ts`**:
    *   Confirm that `downscaleDataUrlToJpeg` uses these constants (already verified).
    *   No logic changes needed here; updating the constants will automatically fix the "save to disk" quality.

This change will ensure that "Retry" attempts use an image just as good as the first attempt, preventing the "Verifier" loops and AI hallucinations caused by blurriness.
