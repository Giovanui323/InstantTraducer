This leaves the backend in a permanently blocked state for that file, causing subsequent save attempts to fail with `BLOCKED_BY_BACKEND`.

Here is the plan to fix this:

1.  **Fix `blockSave` in `src/hooks/useAppLibrary.ts`**:
    *   Modify the `setTimeout` callback within `blockSave` to explicitly call `window.electronAPI.unblockSave(fileId)` when the block duration expires. This ensures the backend is synchronized with the frontend's unblocking.

2.  **Improve Error Handling in `processSaveQueue`**:
    *   Modify `processSaveQueue` in `src/hooks/useAppLibrary.ts` to handle the `BLOCKED_BY_BACKEND` error gracefully.
    *   Instead of throwing a generic error (which results in the "Queued save failed" error log), explicitly check for `BLOCKED_BY_BACKEND`.
    *   Log it as a warning (to reduce noise) and allow the retry logic to handle it, potentially with a specific strategy (e.g., respecting the retry limit but not logging it as a critical failure).
