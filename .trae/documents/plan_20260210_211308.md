### Key Changes:
1.  **Smart Slot Management**: Implemented `acquireSlot` and `releaseSlot` in `useTranslationQueue`.
    -   When a page needs to wait for the previous page's translation (Context Dependency), it now **releases its concurrency slot**.
    -   This allows other independent tasks (e.g., non-sequential pages or other chapters) to start immediately.
    -   When the dependency is resolved, the page **re-acquires a slot** (waiting if necessary) before proceeding to the AI request.

2.  **Memory Optimization**:
    -   Moved the "Wait for Previous Page" logic to the **beginning** of the translation process, *before* the expensive PDF rendering.
    -   This prevents memory saturation: pages waiting for context will not load images into memory until they are actually ready to translate.

3.  **Pipeline Control**:
    -   Updated the dependency check to prevent "Cascading Starts" (where Page 100 waits for 99, which waits for 98...).
    -   Now, a page only starts pipelining (pre-processing) if its previous page is **actively processing** (using a slot). If the previous page is waiting (idling), the next page stays in the queue.
    -   This limits the active pipeline depth to a safe level (Current + Next), ensuring system stability and preventing timeouts.
