# Piano di Refactoring per main.js (Versione Ottimizzata)

Per alleggerire `main.js` senza perdere alcuna funzionalità o dato, sposterò la logica in nuovi moduli specializzati. Riguardo alla tua domanda su `utils`, in `src/utils/` ci sono file per il frontend (TypeScript), mentre per il processo Main (Node.js) creerò un file dedicato per evitare conflitti di dipendenze.

## Nuovi Moduli Proposti

### 1. electron/state.js
Gestione dello stato globale e della concorrenza:
- Esporta la classe `WriteSequencer`.
- Esporta le istanze globali `writeSequencer`, `settingsMutex` e `inconsistencyTracker`.
- Questo garantisce che lo stato sia condiviso correttamente tra tutti i moduli.

### 2. electron/mainUtils.js (per distinguerlo da src/utils)
Funzioni di utilità generiche per il processo Main:
- `withTimeout`, `retry`, `parseImageDataUrl`, `escapeHtml`, `redactSettings`.

### 3. electron/fileUtils.js
Tutta la complessa gestione del file system:
- Funzioni per i percorsi (`getTranslationsDir`, `getTrashDir`, `getAssetsRootDir`, etc.).
- Funzioni di scrittura sicura: `safeWriteFile`, `withFileLock`, `hasEnoughDiskSpace`.
- Logica di ripristino: `recoverTemporaryFiles`, `autoCleanupTrash`.

### 4. electron/projectLogic.js
Logica di business per le operazioni sui progetti:
- `performRename`: Gestione rinomina JSON e cartelle asset.
- `performConsolidation`: Algoritmo di deduplicazione e unione progetti.

## Trasformazione di main.js
Il file `main.js` diventerà molto più snello, occupandosi solo di:
- Importare i nuovi moduli.
- Gestire il ciclo di vita dell'app (`app.on('ready')`, `createWindow`, etc.).
- Registrare gli IPC Handler (`ipcMain.handle`), delegando l'esecuzione alle funzioni nei nuovi moduli.
