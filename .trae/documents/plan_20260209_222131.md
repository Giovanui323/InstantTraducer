### 1. "Kill Switch" for Contaminated Translations
I will modify the application logic to immediately stop subsequent page translations when a page fails its quality check with a "severe" error.

*   **`src/hooks/useAppQuality.ts`**: 
    *   Add an `onSevereError` callback prop.
    *   Trigger this callback when `processVerificationInternal` detects a `severe` error.
*   **`src/App.tsx`**: 
    *   Implement the `onSevereError` handler.
    *   When page `N` fails, identify all currently running translations for pages `> N`.
    *   Forcefully stop these translations using `translation.stopTranslation()`.
    *   Invalidate any potentially finished but contaminated translations in `translationMap` for these pages.

*   **`src/services/translation/TranslationJobRunner.ts`**:
