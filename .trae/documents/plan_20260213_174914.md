Ecco il piano per modificare il comportamento di chiusura dell'applicazione su macOS.

### Analisi del problema
Il comportamento che osservi (l'app rimane nel Dock con il puntino dopo la chiusura della finestra) è dovuto a una specifica configurazione nel file `electron/main.js`.
Attualmente il codice è scritto così:
```javascript
app.on('window-all-closed', function () {
    if (process.platform !== 'darwin') { // Se NON è macOS
        app.quit(); // Chiudi l'app
    }
    // Se è macOS, non fa nulla, mantenendo l'app attiva
});
```
Questo è il comportamento standard raccomandato da Apple per le applicazioni "document-based", ma per utility o tool specifici è spesso preferibile che l'app si chiuda completamente.

### Intervento proposto
Modificherò il file `electron/main.js` per rimuovere il controllo sulla piattaforma, forzando la chiusura dell'applicazione su tutti i sistemi operativi (incluso macOS) quando l'ultima finestra viene chiusa.

#### Passaggi tecnici:
1.  Modificare `electron/main.js`.
2.  Rimuovere la condizione `if (process.platform !== 'darwin')` all'interno dell'evento `window-all-closed`.
3.  Eseguire sempre `app.quit()` quando tutte le finestre sono chiuse.

In questo modo, chiudendo la finestra con la "X" rossa, l'applicazione terminerà completamente il processo e sparirà dal Dock.
