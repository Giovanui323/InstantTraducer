### 1. Fix Race Condition in `WriteSequencer` (`electron/state.js`)

The current implementation of `enqueue` starts the task execution (and its timer) *immediately* upon creation, before chaining it to the previous task. This causes concurrent writes to the same file, leading to lock contention and timeouts.

**Changes:**

* Refactor `enqueue` to ensure the task execution (and any delay) occurs **strictly inside** the Promise chain (`.then()`).

* This guarantees that a new task waits for the previous one to complete before starting, resolving the deadlock.

### 2. Fix Locking in `rename-translation` (`electron/projectHandlers.js`)

The rename operation modifies the file ID, but the new ID is not protected from concurrent writes during the operation. If the UI attempts to save the new ID while rename is in progress, it can cause a race condition.

**Changes:**

* Explicitly call `writeSequencer.block(newFileId)` **before** starting the rename operation.

* Ensure `writeSequencer.unblock(newFileId)` is called in the `finally` block.

* This ensures the new file ID is "reserved" until the rename is fully complete.
