I have analyzed the logs and the codebase, identifying the cause of the "infinite loop" (or rather, a save operation flood) and prepared a fix.

### Problem Analysis
The logs show a flood of `save-translation` requests (increasing count: 21, 1, 2, 3...) happening in parallel with high latency (~3s per save).
1.  **Cause**: In `TranslationExecutor.ts`, the code triggers the save operation asynchronously ("fire and forget") inside the state updater function, without waiting for it to complete.
    ```typescript
    // Current Code
    setters.setTranslationMap(prev => {
      // ... updates state ...
      Promise.resolve().then(() => services.updateLibrary(...)); // Detached promise
      return next;
    });
    ```
2.  **Effect**: The `TranslationJobRunner` moves to the next page immediately after the translation is done, spawning a new save request. Since disk writes are slower than the loop, requests pile up in the Main process queue, causing a backlog and the appearance of an infinite loop.
3.  **Side Issue**: Placing side effects (like network calls or saves) inside a React-style state updater function is an architectural anti-pattern and can lead to unpredictable behavior.

### Proposed Plan
I will refactor `src/services/translation/TranslationExecutor.ts` to enforce sequential execution of save operations.

1.  **Refactor `setTranslationMap`**: Move the `services.updateLibrary` call **out** of the state updater callback.
2.  **Await Operations**: Explicitly `await` the save operation to ensure it completes before the runner proceeds to the next page.
3.  **Await Verification**: Explicitly `await` the `verifyAndMaybeFixTranslation` call to prevent verification jobs from overlapping or flooding the queue.

### Code Changes
**File**: `src/services/translation/TranslationExecutor.ts`

```typescript
// Step 1: Update state synchronously
setters.setTranslationMap(prev => ({ ...prev, [targetPage]: normalizedText }));

// Step 2: Perform save sequentially (and await it)
if (metadata) {
  await services.updateLibrary(metadata.name, {
    ...(currentProjectFileId ? { fileId: currentProjectFileId } : {}),
    translations: { [targetPage]: normalizedText },
    translationsMeta: { [targetPage]: metaUpdate },
    annotations: { [targetPage]: result.annotations || [] }
  });
}

// ...

// Step 3: Await verification
await services.verifyAndMaybeFixTranslation({ ... });
```

This change effectively throttles the background job to the speed of your disk I/O, preventing the "infinite loop" flood while ensuring data integrity.
