I have identified the cause of the infinite loop:
1. The `updateLibrary` function in `useAppLibrary.ts` depends on `recentBooks`, so it is recreated every time a save occurs (which updates `recentBooks`).
2. In `App.tsx`, the `useEffect` that saves `lastPage` (lines 463-467) depends on `library.updateLibrary`.
3. This creates a cycle: `updateLibrary` called -> `recentBooks` updates -> `updateLibrary` recreated -> `useEffect` triggers -> `updateLibrary` called again.

**Plan:**
1.  **Modify `src/hooks/useAppLibrary.ts`**:
    *   Remove `recentBooks` and `currentProjectFileId` from the dependency array of `updateLibrary`. The function already uses `recentBooksRef` and `currentProjectFileIdRef` internally, so these state dependencies are unnecessary and cause the instability.

This will make `updateLibrary` stable, breaking the infinite loop.
