# Piano di Intervento: Ottimizzazione Cestino e Pulizia Automatica

Ho analizzato il codice e confermo pienamente la tua analisi. Hai individuato un punto critico: la logica di cancellazione ﾃｨ duplicata, fragile e non pianificata.

Ecco i dettagli tecnici riscontrati:
1.  **Discrepanza di Parsing:** 
    *   Il backend (`fileUtils.js`) fa un parsing manuale e fragile: `item.split('_')[1]`.
    *   Le API (`projectHandlers.js`) usano correttamente l'helper `parseTrashFolderName` da `idUtils.js`.
    *   **Rischio:** Se il formato della cartella cambia, il backend smette di pulire senza dare errori, mentre la UI potrebbe continuare a mostrare i file.
2.  **Mancanza di Pianificazione:** La pulizia avviene solo all'avvio (`app.on('ready')`). Se l'app rimane aperta per giorni, i file scaduti non vengono rimossi.
3.  **Logica Duplicata:** Il calcolo dei "giorni rimanenti" ﾃｨ replicato nel frontend (React) e nel backend, creando potenziali disallineamenti visivi (es. la UI dice "1 giorno" ma il backend lo cancella subito).

## Obiettivi
1.  **Unificare la logica:** Centralizzare il calcolo della scadenza.
2.  **Automatizzare:** Eseguire la pulizia periodicamente senza richiedere il riavvio.
3.  **Robustezza:** Usare parser condivisi e meccanismi di retry.

## Passaggi Operativi

### 1. Refactoring Backend (`electron/fileUtils.js`)
Modificherﾃｲ `autoCleanupTrash` per:
*   Importare e utilizzare `parseTrashFolderName` da `idUtils.js` (invece di fare `split('_')` manuale).
*   Implementare un meccanismo di **Retry** per la cancellazione (usando la funzione `retry` esistente o un loop locale) per gestire file bloccati da altri processi.
*   Aggiungere log piﾃｹ dettagliati sui file saltati o sugli errori.

### 2. Sincronizzazione UI/Backend (`electron/projectHandlers.js`)
Aggiornerﾃｲ l'handler `get-trash-contents`:
*   Invece di restituire solo `deletedAt`, restituirﾃ anche `expiresAt` o `daysLeft` calcolato direttamente dal backend.
*   Questo renderﾃ la UI "stupida": dovrﾃ solo visualizzare il valore fornito dal backend, garantendo che ciﾃｲ che vedi ﾃｨ esattamente ciﾃｲ che accadrﾃ.

### 3. Pianificazione Automatica (`electron/main.js`)
*   Aggiungerﾃｲ un `setInterval` (es. ogni 12 o 24 ore) per invocare `autoCleanupTrash` anche mentre l'app ﾃｨ in esecuzione.
*   Questo garantisce che le risorse vengano liberate anche in sessioni di lavoro lunghe (server/kiosk mode).

### 4. Pulizia Metadata (Opzionale ma raccomandato)
*   Verificherﾃｲ che la cancellazione fisica rimuova correttamente eventuali riferimenti residui. Attualmente il sistema sposta l'intera cartella, quindi il rischio di orfani ﾃｨ basso, ma aggiungerﾃｲ un controllo di sicurezza.

Se confermi, procederﾃｲ con queste modifiche iniziando dal refactoring di `fileUtils.js`.
