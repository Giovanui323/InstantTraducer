I have analyzed the code and found the cause of the issue.

The problem is in `src/components/OriginalPreviewModal.tsx`.
1.  **Centering Logic**: The modal uses `flex items-center justify-center` to center the content. When the content (PDF or image) is zoomed in and becomes larger than the screen, this centering method causes the left and top parts of the content to be "clipped" and inaccessible via scrolling.
2.  **Image Zooming**: If you are viewing an image (fallback mode), it uses CSS `transform: scale(...)`. This scales the image visually but does not update the layout size, preventing scrollbars from appearing.

**Plan to fix:**
1.  **Fix Centering**: Modify `OriginalPreviewModal.tsx` to use `margin: auto` on the content container instead of `justify-center` on the parent. This ensures that when the content is smaller than the screen, it is centered, but when it is larger, it aligns to the start, allowing full scrolling.
2.  **Fix Image Zoom**: Change the image rendering to use the `zoom` CSS property (supported in Electron) instead of `transform`. This ensures the layout size updates correctly when zooming, triggering the scrollbars.
